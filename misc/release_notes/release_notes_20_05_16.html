RELEASE NOTES FOR KOHA 20.05.16

23 Sep 2021

Koha is the first free and open source software library automation package (ILS). Development is sponsored by libraries of varying types and sizes, volunteers, and support companies from around the world. The website for the Koha project is:

Koha 20.05.16 is a bugfix/maintenance release with security fixes.

It includes 6 security fixes, 1 enhancements, 3 bugfixes.

System requirements

You can learn about the system components (like OS and database) needed for running Koha here: https://wiki.koha-community.org/wiki/System_requirements_and_recommendations

Security bugs

Koha

  • [28759] Users with pretty basic staff interface permissions can see/add/remove API keys of any other user
  • [28772] Any user that can work with reports can see API keys of any other user
  • [28929] No filtering on borrowers.flags on member entry pages (OPAC, self registration, staff interface)
  • [28935] No filtering on patron's data on member entry pages (OPAC, self registration, staff interface)
  • [28941] No filtering on suggestion at the OPAC
  • [28947] OPAC user can create new users

Enhancements

OPAC

  • [26847] Make borrower category code accessible in all pages of the OPAC

Critical bugs fixed

OPAC

  • [28885] OpacBrowseResults can cause errors with bad search indexes

Other bugs fixed

Staff Client

  • [28722] tools/batchMod.pl needs to import C4::Auth::haspermission
  • [28802] Untranslatable strings in browser.js
